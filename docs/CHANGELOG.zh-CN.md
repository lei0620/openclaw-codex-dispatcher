# 更新记录

## v1.9.3 - 多窗口精确刷新版

- 两个 Codex 窗口即使共用同一个进程，也会按真实窗口句柄精确刷新，不再把“同 PID”误认为“同窗口”。
- 刷新手机绑定的 Codex 窗口后，会恢复用户原来正在操作的前台窗口，减少打断其他并行工作。
- 电脑存在多个 Codex 窗口时，手机新对话和已有对话都必须先绑定目标窗口，禁止系统猜测。
- “自动切到桌面最新对话”改为默认关闭；只有用户主动开启后，定时校准才会改变当前会话，已有设置保持不变。

## v1.9.2 - 会话隔离修正版

- 快速切换项目或对话时，旧会话较慢返回的网络响应不会再覆盖当前聊天内容。
- 手机消息发送成功后，如果用户已经切到其他对话，不会再被响应强制拉回原对话。
- 相同文字的历史消息与手机任务改为一对一配对；重复发送“在吗”等短消息时，新的气泡不会被旧历史吞掉或完成后消失。
- NAS 在任务执行、取消或等待授权时重启，会把已经中断的任务明确收口并释放会话/窗口队列，不再永久卡在“执行中”。
- 重启恢复结果会追加到实时事件流，手机重连后立即更新任务状态，不再等待定时刷新。

## v1.9.1 - 后台恢复修正版

- 修复后台提醒服务被系统停止后，重新打开 App 没有根据已保存开关自动恢复连接的问题。
- 恢复服务前会重新检查系统通知权限；权限已关闭时不会静默启动一个无法显示通知的后台服务。
- 开机恢复和 App 前台恢复共用同一套安全检查，避免状态显示开启但服务实际未运行。
- 权限请求被批准或拒绝后，锁屏通知会立即撤销，不再保留已经失效的“等待授权”提醒。
- 修复荣耀等机型在锁屏时仍短暂把 Activity 标记为前台，导致权限通知被错误静默的问题。
- 设置页显示后台连接的真实状态和最后事件 ID，能够区分正在连接、认证失败、网络错误和已连接。
- 开启后台提醒后会持有可随开关释放的 CPU 与 Wi-Fi 保活锁，改善荣耀等机型锁屏后冻结实时连接的问题；关闭功能时立即释放。
- 修复实时通道刚重连时握手误用旧连接引用，导致偶发在线状态不稳定的问题。

## v1.9.0 - 后台提醒版

- 可在设置中主动开启后台提醒，锁屏或切到其他 App 后继续接收 Codex 权限请求、任务完成和失败通知。
- 后台连接使用独立的 NAS 实时通道和事件游标，不会替用户批准、拒绝、取消或重复执行任务。
- App 正在前台时自动抑制系统弹窗，避免聊天页和通知栏重复显示同一条结果。
- 开启状态会保留；手机重启或 App 更新后自动恢复，也可以随时在设置中关闭。
- NAS 服务地址和访问密码继续由 Android Keystore 加密保存，后台连接不会把密码写入网址或日志。

## v1.8.0 - 安全存储版

- Android 上的 NAS 服务地址和访问密码改由 Android Keystore AES-GCM 加密保存，不再留在 WebView localStorage。
- 从旧版本升级时自动迁移现有地址和密码，迁移成功后删除旧明文，无需重新输入。
- 原生加密读取或写入失败时明确提示并保留旧数据，不会静默清空凭据。
- 浏览器访问 NAS 面板时继续使用浏览器本地设置，不影响网页版使用。

## v1.7.0 - 交互恢复版

- App 从后台回到前台、网络恢复或页面重新显示时，自动重连实时通道并校准状态，不会自动重发任务。
- 用户向上阅读历史时，新回复不再强制跳到底部；显示“回到最新”按钮供手动返回。
- 会话列表增加“执行中”“等授权”“排队”“失败”状态标记，切换并行任务更直观。
- 设置页增加连接诊断，可查看 NAS 延迟、实时连接、Win11 心跳、Codex 状态和当前会话映射。
- 一键复制诊断时自动隐藏 Token、密码、Authorization、WebDAV 凭据和密钥格式内容。

## v1.6.0 - 实时消息版

- 手机通过 NAS 专用 WebSocket 通道实时接收任务状态、回复、权限请求和电脑在线变化。
- 网络断开后按事件游标自动补发，游标过期时自动完整校准，不再依赖高频整页刷新。
- 手机发送后立即显示消息气泡；失败时可在原消息上重试，相同消息不会重复执行。
- 同一个对话或同一个绑定窗口串行排队，不同未绑定对话可以并行执行。
- 完整刷新由每 2 秒降低为每 30 秒兜底，减少滚动跳动、重复消息和 NAS 请求量。

## v1.5.0 - 稳定连接版

- 顶部连接图标可查看 NAS、Win11 电脑代理和 Codex 服务三层真实状态。
- 兼容新版 `ChatGPT.exe` 和旧版 `Codex.exe` 桌面宿主，并排除辅助小窗口和命令行进程。
- Win11 代理会预热并持续检查 Codex 会话服务，异常后自动恢复。
- 手机会话固定走对应的 Codex 会话通道，服务不可用时明确失败，不再回退到可能发错频道的方式。
- 关闭 Android 原生桥接请求日志，避免访问凭据出现在调试日志中。

## v1.3.0 - 自动激活版

- 飞牛 NAS 重启后自动执行 `docker compose up -d`，恢复手机面板服务。
- Win11 登录后自动常驻监听器，打开 Codex 客户端后代理自动上线。
- 手机设置页显示当前版本和本版更新内容。
- 保持 GitHub 源码去敏：不提交真实访问密码、WebDAV 密码、本地配置和 APK 历史包。
