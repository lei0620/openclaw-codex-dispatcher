# 更新记录

# v1.9.19 - 运行和未读状态同步

- 电脑端直接开始的 Codex 任务也会显示在手机顶部“正在执行 / 待查看”卡片中。
- 任务完成后自动转为未读结果卡片，后台运行时不再被误判为已读。
- 同步电脑端对话查看状态；电脑已经查看的结果不会再在手机重复提醒。
- 保留 v1.9.18 已确认的现代蓝色闪电图标。

# v1.9.18 - 现代蓝色闪电图标

- App 图标改为居中的小尺寸蓝色闪电，增加四周留白。
- 移除 Wi-Fi 图形、塑料高光、立体浮雕和厚重阴影，采用现代扁平风格。
- 同步更新普通圆角、圆形兼容和 Android 自适应图标资源。

# v1.9.17 - 蓝白色 Wi-Fi 图标

- App 换用纯蓝白色 iOS 风格 Wi-Fi 图标，移除设备组合和绿色状态点。
- 普通圆角、圆形兼容和 Android 自适应资源均使用同一 Wi-Fi 视觉。
- 作为独立新版本发布，已安装短暂试用版的手机也能正常检测更新。

# v1.9.16 - 设备联动图标试用版

- App 换用全新生成式启动图标，以手机、电脑和在线绿点表达远程控制。
- 普通圆角、圆形兼容和 Android 自适应图标均已重新生成，适配不同桌面启动器。
- 保留 1254×1254 生成原稿，后续可继续制作商店图或品牌素材。

# v1.9.15 - 桌面会话列表一致性

- 手机每个项目最近 5 个对话改为按 Codex 桌面端最近活跃时间排序。
- 已归档会话和 Codex 内部工作线程不再混入手机普通对话列表。
- NAS 会按 Win11 最新会话快照清理无效旧记录；已有手机任务和结果的会话会安全保留。
- Win11 只解析最终入选的会话文件，减少后台同步耗时和对实时消息的影响。

# v1.9.14 - 消息实时恢复加固

- Android 从锁屏、后台或系统暂停状态恢复时，立即重连实时通道并补齐消息。
- 使用原生 App 生命周期信号补充浏览器恢复事件，减少偶发延迟和手动刷新。
- 保留原生后台实时连接；点击“彻底退出”后仍会停止所有后台服务。

# v1.9.13 - 未读结果与彻底退出

- 其他项目或其他对话的任务完成、失败后，会继续显示在顶部“正在执行 / 待查看”区域。
- 点击未读结果卡片进入对应对话后，卡片自动清除；App 重启后仍能恢复未读状态。
- 修复 Android 前台 WebView 拦截局域网实时连接、必须刷新才看到结果的问题。
- 右上角设置按钮旁新增“彻底退出 Codex”，可停止实时连接、后台提醒和保活服务，并彻底关闭 App。

## v1.9.12 - 最近对话扩展版

- 每个项目由同步最近 3 个 Codex 对话调整为最近 5 个。
- 手机侧栏优先读取带真实桌面 session 的 Codex 对话，空白草稿和仅面板测试会话不再挤占 5 个位置。
- 手机首次加载仍按项目限量获取，不会一次加载全部历史。
- 顶部“最近使用”继续显示最近 3 个不同项目，避免侧栏顶部过长。

## v1.9.11 - 回答内容净化版

- 手机聊天不再显示 `git-stage`、`git-commit`、`git-push` 等 Codex 桌面内部操作标记。
- 同步到 NAS 时先清理内部标记，手机显示前再次过滤，已有历史消息也能保持干净。
- 正常回答、代码内容和折叠的处理过程保持不变。

## v1.9.10 - 实时消息稳定版

- 前台实时通道会在局域网与 Tailscale/VPN 地址之间自动切换，不再只对着失效地址反复重连。
- 某个地址 5 秒内无法建立实时连接时自动尝试下一条路径，缩短网络切换后的等待时间。
- 实时通道离线时每 5 秒自动校准一次状态，作为低频安全网；连接正常时仍由 WebSocket 低延迟推送。
- 保留后台原生服务的前台保活、15 秒心跳、断线重连和事件游标补齐能力。

## v1.9.9 - 最终答案与处理过程分离版

- 保留 Codex 桌面会话中的 commentary 与 final_answer 阶段，不再把处理中内容当作普通回答。
- 处理过程在任务完成后默认折叠，最终答案单独显示；任务尚未完成时可以展开查看实时进度。
- 修复处理过程与最终答案文字相同或时间接近时可能被错误去重的问题。

## v1.9.8 - 手机与桌面会话同步修复版

- 手机任务开始后立即刷新绑定的 Codex 桌面窗口，让手机消息更快出现在电脑对应会话中。
- 手机为每条已发送任务单独追踪 NAS 状态，实时事件偶尔断开时也能及时拿到完成、失败和授权状态。
- 单任务追踪按任务 ID 隔离，不会切错对话，也不会影响其他窗口并行执行。

## v1.9.7 - Codex 移动端导航优化版

- 顶部品牌固定显示 Codex，菜单、窗口、设备、同步、权限和设置统一使用全新生成式图标。
- 左侧栏优先显示正在执行的对话和最近三个不同项目，再展示完整项目与历史对话。
- 点击最近项目直接进入该项目最新会话，继续沿用原有精准窗口绑定，不改变并行任务规则。
- 修复存在运行任务时顶部运行区被拉高、挤压聊天内容的问题，手机与桌面宽度都保持紧凑。

## v1.9.6 - 更新检查超时保护版

- 包含 v1.9.5 的实时同步降噪、重复 session 合并和 NAS 写入优化。
- 更新清单使用 8 秒连接、12 秒读取超时，网络异常时不再让“检查更新”长时间无响应。
- APK 下载继续保留较长读取时间，不影响较慢网络下的完整下载。
- 更新服务器超时时显示明确的中文重试提示。

## v1.9.5 - 实时同步降噪版

- 合并属于同一个 Codex session 的多个 rollout 文件，避免一个对话被重复上报并在不同历史版本间来回覆盖。
- NAS 对重复会话上报做二次合并，同样内容再次同步时不再产生手机事件。
- 没有变化的 Codex 同步不再重写 NAS 状态文件，降低无效磁盘 I/O。
- 高频流式日志在 80ms 内合并，只更新聊天区域；完成、失败、授权等关键状态仍立即显示。
- 保留局域网与 Tailscale/VPN 自动切换能力。

## v1.9.4 - 局域网与 VPN 自动切换版

- 手机在家时优先使用 NAS 局域网地址，连接失败后自动尝试原 Tailscale/VPN 地址。
- App 切换到可用地址后会记住当前路径，并同步重启前台实时连接。
- Android 后台提醒会在局域网和 VPN 地址之间自动轮换重连。
- 只有读取请求和带唯一消息编号的任务发送允许自动换路重试，避免重复创建对话或重复执行高风险操作。
- 密码错误、权限错误等服务端拒绝不会触发换路，便于准确显示真实问题。

## v1.9.3 - 多窗口精确刷新版

- 两个 Codex 窗口即使共用同一个进程，也会按真实窗口句柄精确刷新，不再把“同 PID”误认为“同窗口”。
- 刷新手机绑定的 Codex 窗口后，会恢复用户原来正在操作的前台窗口，减少打断其他并行工作。
- 电脑存在多个 Codex 窗口时，手机新对话和已有对话都必须先绑定目标窗口，禁止系统猜测。
- “自动切到桌面最新对话”改为默认关闭；只有用户主动开启后，定时校准才会改变当前会话，已有设置保持不变。

## v1.9.2 - 会话隔离修正版

- 快速切换项目或对话时，旧会话较慢返回的网络响应不会再覆盖当前聊天内容。
- 手机消息发送成功后，如果用户已经切到其他对话，不会再被响应强制拉回原对话。
- 相同文字的历史消息与手机任务改为一对一配对；重复发送“在吗”等短消息时，新的气泡不会被旧历史吞掉或完成后消失。
- NAS 在任务执行、取消或等待授权时重启，会把已经中断的任务明确收口并释放会话/窗口队列，不再永久卡在“执行中”。
- 重启恢复结果会追加到实时事件流，手机重连后立即更新任务状态，不再等待定时刷新。

## v1.9.1 - 后台恢复修正版

- 修复后台提醒服务被系统停止后，重新打开 App 没有根据已保存开关自动恢复连接的问题。
- 恢复服务前会重新检查系统通知权限；权限已关闭时不会静默启动一个无法显示通知的后台服务。
- 开机恢复和 App 前台恢复共用同一套安全检查，避免状态显示开启但服务实际未运行。
- 权限请求被批准或拒绝后，锁屏通知会立即撤销，不再保留已经失效的“等待授权”提醒。
- 修复荣耀等机型在锁屏时仍短暂把 Activity 标记为前台，导致权限通知被错误静默的问题。
- 设置页显示后台连接的真实状态和最后事件 ID，能够区分正在连接、认证失败、网络错误和已连接。
- 开启后台提醒后会持有可随开关释放的 CPU 与 Wi-Fi 保活锁，改善荣耀等机型锁屏后冻结实时连接的问题；关闭功能时立即释放。
- 修复实时通道刚重连时握手误用旧连接引用，导致偶发在线状态不稳定的问题。

## v1.9.0 - 后台提醒版

- 可在设置中主动开启后台提醒，锁屏或切到其他 App 后继续接收 Codex 权限请求、任务完成和失败通知。
- 后台连接使用独立的 NAS 实时通道和事件游标，不会替用户批准、拒绝、取消或重复执行任务。
- App 正在前台时自动抑制系统弹窗，避免聊天页和通知栏重复显示同一条结果。
- 开启状态会保留；手机重启或 App 更新后自动恢复，也可以随时在设置中关闭。
- NAS 服务地址和访问密码继续由 Android Keystore 加密保存，后台连接不会把密码写入网址或日志。

## v1.8.0 - 安全存储版

- Android 上的 NAS 服务地址和访问密码改由 Android Keystore AES-GCM 加密保存，不再留在 WebView localStorage。
- 从旧版本升级时自动迁移现有地址和密码，迁移成功后删除旧明文，无需重新输入。
- 原生加密读取或写入失败时明确提示并保留旧数据，不会静默清空凭据。
- 浏览器访问 NAS 面板时继续使用浏览器本地设置，不影响网页版使用。

## v1.7.0 - 交互恢复版

- App 从后台回到前台、网络恢复或页面重新显示时，自动重连实时通道并校准状态，不会自动重发任务。
- 用户向上阅读历史时，新回复不再强制跳到底部；显示“回到最新”按钮供手动返回。
- 会话列表增加“执行中”“等授权”“排队”“失败”状态标记，切换并行任务更直观。
- 设置页增加连接诊断，可查看 NAS 延迟、实时连接、Win11 心跳、Codex 状态和当前会话映射。
- 一键复制诊断时自动隐藏 Token、密码、Authorization、WebDAV 凭据和密钥格式内容。

## v1.6.0 - 实时消息版

- 手机通过 NAS 专用 WebSocket 通道实时接收任务状态、回复、权限请求和电脑在线变化。
- 网络断开后按事件游标自动补发，游标过期时自动完整校准，不再依赖高频整页刷新。
- 手机发送后立即显示消息气泡；失败时可在原消息上重试，相同消息不会重复执行。
- 同一个对话或同一个绑定窗口串行排队，不同未绑定对话可以并行执行。
- 完整刷新由每 2 秒降低为每 30 秒兜底，减少滚动跳动、重复消息和 NAS 请求量。

## v1.5.0 - 稳定连接版

- 顶部连接图标可查看 NAS、Win11 电脑代理和 Codex 服务三层真实状态。
- 兼容新版 `ChatGPT.exe` 和旧版 `Codex.exe` 桌面宿主，并排除辅助小窗口和命令行进程。
- Win11 代理会预热并持续检查 Codex 会话服务，异常后自动恢复。
- 手机会话固定走对应的 Codex 会话通道，服务不可用时明确失败，不再回退到可能发错频道的方式。
- 关闭 Android 原生桥接请求日志，避免访问凭据出现在调试日志中。

## v1.3.0 - 自动激活版

- 飞牛 NAS 重启后自动执行 `docker compose up -d`，恢复手机面板服务。
- Win11 登录后自动常驻监听器，打开 Codex 客户端后代理自动上线。
- 手机设置页显示当前版本和本版更新内容。
- 保持 GitHub 源码去敏：不提交真实访问密码、WebDAV 密码、本地配置和 APK 历史包。
