# Codex 移动端顶部与侧栏优先级改版设计

状态：已确认视觉方向，待用户确认书面规格后进入实现。

## 目标

- 聊天页顶部品牌固定显示 `Codex`，不再用当前项目名代替产品名。
- 使用一套统一生成的透明背景图标，替换现有风格不一致的顶部图标。
- 左侧项目选择界面优先展示正在执行的对话，以及最近三次对话对应的项目。
- 保留窗口绑定、并行任务、权限处理、实时同步、局域网与 VPN 自动切换等已有功能。

## 不在本次范围内

- 不修改 NAS 与 Win11 代理的接口、数据结构和鉴权方式。
- 不增加项目分类功能。
- 不改变桌面输入模拟、Codex 会话绑定和同窗口串行排队规则。
- 不重新设计聊天消息、输入框和设置页。

## 视觉方向

采用克制的黑灰色与翡翠绿点缀。界面保持轻量、清楚、工作导向，不使用装饰性大卡片、渐变球或复杂背景。

图标采用生成式位图素材，而不是临时字符或手绘 SVG：

- 每个图标为独立透明 PNG，源文件尺寸 `192 x 192`。
- 实际显示尺寸为 `22-24px`，在固定尺寸按钮中居中，切换状态不得引起布局位移。
- 图形视觉线宽统一约 `2px`，主色接近黑色，激活状态使用翡翠绿。
- 生成菜单、窗口、设备状态、同步、权限、设置六个图标。
- 图标按钮保留无障碍名称和提示文字，移除移动端蓝色点击高亮。

## 顶部栏

- 主标题固定为 `Codex`。
- 如需显示当前会话名称，放在主标题下方作为紧凑的次级信息，单行截断，不挤压右侧按钮。
- 按钮功能顺序保持为：菜单、窗口、连接状态、同步、权限、设置。
- 手机窄屏必须完整显示所有按钮；必要时压缩按钮间距和标题最大宽度，不隐藏关键按钮。
- 顶部栏保持吸顶，并正确避开全面屏状态栏安全区域。

## 左侧栏结构

侧栏从上到下固定为以下顺序：

1. `正在执行`
2. `最近使用`
3. `全部项目`

### 正在执行

- 使用现有 `state.activeTasks` 数据和现有 `compareActiveTasks` 排序规则。
- 每一项显示运行状态、会话名称和所属项目，避免同名窗口无法分辨。
- 点击后切换到对应会话，并继续沿用该会话已有的精确窗口绑定。
- 不改变不同窗口并行、同窗口串行排队的规则。
- 没有运行任务时隐藏整个区域，不显示空卡片。

### 最近使用

- 从 `state.conversations` 按 `updatedAt` 从新到旧排序。
- 按 `projectId` 去重，只保留最近使用的三个不同项目。
- 一个项目存在多个近期会话时，点击项目默认进入该项目更新时间最新的会话。
- 已删除、缺失或无法匹配 `state.projects` 的项目跳过，不显示失效入口。
- 没有有效历史会话时隐藏整个区域。

### 全部项目

- 沿用当前完整项目和会话列表的展示与点击逻辑。
- 最近使用区域出现过的项目仍可在全部项目中保留，方便用户查看该项目的其他历史会话。
- 本次不加入自定义项目分类，也不改变每个项目最近三次对话的加载限制。

## 数据边界

本次排序和分组只从已有的以下状态派生，不新增后端字段：

- `state.projects`
- `state.conversations`
- `state.activeTasks`

最近项目列表应通过独立纯函数生成，保证排序、去重和缺失项目处理可以单独测试。

## 交互与兼容

- 切换正在执行的对话或最近项目后，手机发送目标必须立即更新为所选会话，不能继续发到此前窗口。
- 切换只影响当前手机所选会话，不刷新或打断其他 Codex 窗口正在运行的任务。
- 权限角标、权限卡片、同步状态、在线状态和设置入口保持原有行为。
- 聊天滚动位置、底部输入框和实时流式消息更新不得因侧栏改版发生回归。
- 局域网优先与 Tailscale/VPN 外网备用连接策略保持不变。

## 验收标准

- 顶部固定显示 `Codex`，窄屏下六个功能图标完整可见且不重叠。
- 图标为同一视觉体系的透明 PNG，普通态与激活态容易分辨。
- 侧栏顶部顺序正确：正在执行、最近使用、全部项目。
- 最近使用最多显示三个不同项目，顺序与最近会话更新时间一致。
- 点击运行会话、最近项目和全部项目中的会话，发送目标均与当前选择一致。
- 多个 Codex 窗口同时运行时，切换一个会话不会打断其他窗口。
- 输入框仍固定在聊天页底部，顶部栏仍吸顶，全面屏安全区域正常。

## 验证计划

- 为最近项目派生函数增加排序、去重、三项上限和缺失项目测试。
- 增加界面静态测试，检查固定 `Codex` 标题、图标资源和三个侧栏区域。
- 在手机宽度和桌面宽度各做一次浏览器截图检查。
- 在真机检查顶部适配、侧栏切换、窗口绑定、权限、同步和设置按钮。
- 回归验证实时消息、任务排队、局域网连接和 VPN 备用连接。

## 发布方式

实现和验证通过后再提升版本号。发布流程继续沿用现有方式：清理打包目录、使用 JDK 21 构建、无线调试覆盖安装、旧 APK 归档后发布到 WebDAV、备份并部署 NAS 前端资源，最后提交并推送 Git 仓库。
