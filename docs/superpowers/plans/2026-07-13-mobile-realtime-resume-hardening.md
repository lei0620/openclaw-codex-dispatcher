# Mobile Realtime Resume Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restore low-latency mobile messages after Android backgrounding or lock-screen suspension without requiring manual refresh.

**Architecture:** Keep the native background WebSocket as the background delivery path and add the official Capacitor App lifecycle as a reliable foreground-resume signal. Feed native and browser lifecycle signals into the existing deduplicated recovery controller, which restarts realtime and performs one guarded reconciliation.

**Tech Stack:** JavaScript ES modules, Vitest, Capacitor 8 Android, `@capacitor/app`, Gradle, ADB.

## Global Constraints

- Preserve full exit: it must continue disabling the background service until the user explicitly enables it again.
- Do not add high-frequency polling.
- Do not change task dispatch, window binding, message ordering, or unread-card behavior.
- Use TDD for lifecycle behavior.
- Release as Android version `1.9.14`, version code `55`.
